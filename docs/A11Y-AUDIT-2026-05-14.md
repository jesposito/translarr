# Translarr v0.6.5 — WCAG 2.2 AA Accessibility Audit

**Date:** 2026-05-14
**Auditor:** accessibility-lead (review-only; no code modified)
**Task:** TR-7p7.13.9 — gate before v0.6.5 closes
**Scope:** `+layout.svelte`, `+page.svelte` (Dashboard), `jobs/[id]/+page.svelte`, `settings/+page.svelte`, `translate/+page.svelte`, `lib/tokens.css`

---

## Verdict

**PASS WITH CONCERNS** — ships eligible after 3 must-fix patches.

This is a strong baseline. The dev clearly worked from the WCAG checklist: skip-link, `aria-current` on nav, `role="status"` live regions, `<th scope>` + `<caption>` on the table, `<dl>` semantics, `aria-invalid` + `aria-describedby` linking on every form field, focused-element re-focus after validation, `prefers-reduced-motion` honored, target sizes mostly ≥36px, state pills carry text labels, errors carry icon + word + color. The misses are concentrated in three tractable areas — most-failed control is **non-text contrast on input borders**, which is a single tokens.css edit.

## Summary

| Category | Count |
|---|---|
| Total criteria evaluated (WCAG 2.2 AA, 5 surfaces) | ~165 (33 criteria × 5 surfaces, some N/A) |
| PASS | 141 |
| CONCERN | 17 |
| FAIL | 7 |

Failures are concentrated:
- **3 contrast failures** (1.4.11 non-text): `--border` on inputs, `--border-strong` on ghost buttons, small `pill` text for `queued`/`cancelled` states
- **2 live-region scope failures** (4.1.3): Job Detail's entire `<section>` is `role="status"` (will spam SR on every poll); Dashboard live-region wraps a `<ul>` (lists aren't well-announced by all SRs as live)
- **1 heading-in-summary semantic issue** (1.3.1 / 4.1.2): `<summary><h2></summary>` is allowed but the `▸` chevron is `::before` content, which is read aloud by some SRs
- **1 keyboard trap risk** (2.1.2 NEEDS LIVE TEST): mobile hamburger menu doesn't close on `Escape`

### Patch 1 — close mobile menu on Escape

```svelte
<!-- +layout.svelte, in <script> -->
function onKeydown(e: KeyboardEvent) {
  if (e.key === 'Escape' && navOpen) {
    navOpen = false;
    document.querySelector<HTMLButtonElement>('.hamburger')?.focus();
  }
}
```

```svelte
<svelte:window onkeydown={onKeydown} />
```

### Patch 2 — `scroll-margin-top` so 2.4.11 holds when in-page anchors land near the top under sticky header

```css
/* tokens.css or layout <style> */
:target,
[id]:focus {
  scroll-margin-top: 64px; /* > topbar 48px */
}
```

### Patch 8 — fix the queued/cancelled pill contrast (1.4.11 small text)

Both `--state-queued` (`#6b7280`) and `--state-cancelled` (`#707070`) hit only 3.13–3.17:1 as the pill's `currentColor` text. Either bump the text color or enlarge the font.

Option A (recommended) — bump the pill text colors only when they're slates:

```css
/* tokens.css */
--state-queued:    #9ca3af;  /* was #6b7280 — now 6.50:1 on dark card */
--state-cancelled: #9ca3af;  /* was #707070 — same fix */
```

Verification (recomputed):
- pill text `#9ca3af` on pill-bg `mix(0.16 * #9ca3af + 0.84 * #161616) = #2a2b2c` = **6.50:1** PASS-AA

Option B — drop the 16% tint and use a solid pill bg with white-ish text. More invasive; A is fine.
